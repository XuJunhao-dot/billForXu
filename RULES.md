# billForXu 项目约束（Rules）

> 这些规则优先级高于个人偏好；后续每次迭代都以此为准。

## 1. 多端适配（硬性要求）
- **所有页面必须支持移动端展示（mobile-first）**：部署后需在手机端查看与使用。
- 默认以 **375px 宽度**设计与自测（iPhone 常见宽度），再扩展到桌面端。
- 交互与布局要求：
  - 关键操作按钮在移动端可触达，避免被底部工具条/安全区遮挡。
  - 列表/表格在移动端需可读：必要时使用卡片化、分组、或横向滚动。
  - 图表在移动端需可用：简化图例/缩放/横向滚动/替代摘要列表，避免挤压导致不可读。
  - 表单输入优先移动端体验：使用合适的 input type（date/time/number/decimal），减少手输与误差。

## 2. 单用户免登录（当前阶段）
- 默认单用户免登录；不做账号系统与复杂权限。

## 3. 数据可信与可恢复
- 禁止使用浮点存钱；金额处理必须可控（两位小数/或整数分）。
- 写入必须尽量避免重复提交造成重复记录（幂等/去重）。

## 4. 变更原则
- 每次新增/修改页面都需要在移动端视口下过一遍（至少 375px）。
- 发现移动端不可用视为 P0 缺陷，优先修复。
