# 个人财务记录网站 PRD v0.1（MVP）
面向：手机 + 电脑可用的响应式 Web（不做原生 App）  
MVP 功能：  
1) 记录财务状态（资产/负债，支持资产类型、多级分类）  
2) 图表展示每次输入的趋势（随快照累积）

---

## 1. 目标（Goals）
### 1.1 产品目标
- 让用户用**最少步骤**记录“此刻我有多少资产/负债”，长期形成可视化趋势。
- 支持资产/负债的**结构化管理**（类型 + 多级分类），便于复盘与对比。

### 1.2 成功指标（MVP）
- 新用户在 3 分钟内完成：创建至少 1 条资产 + 1 条负债，并生成第 1 次“财务快照”。
- 7 日内留存：用户至少记录 2 次快照，并查看趋势图 ≥1 次。
- 关键流程转化率：  
  - 进入“新增快照”页 → 成功保存快照 ≥ 60%  
  - 保存快照 → 查看趋势图 ≥ 40%

---

## 2. 范围（Scope）
### 2.1 In Scope（本期做）
- 用户创建“财务快照”（Snapshot）：一次录入包含多条资产/负债明细。
- 资产/负债明细支持：金额、币种（先单币种）、账户/标的名称、类型、**多级分类**、备注。
- 多级分类体系：支持自定义（至少 2 级，最多 4 级）。
- 快照列表：按时间倒序展示；可查看单次快照详情。
- 趋势图：  
  - 总资产趋势、总负债趋势、净资产趋势（净资产=总资产-总负债）  
  - 可按时间范围：近 1 月/3 月/6 月/1 年/全部
- 基础数据校验与错误提示。
- 响应式布局（移动端优先）。

### 2.2 Out of Scope（本期不做）
- 现金流记账（收入/支出流水）、预算、账单导入、银行/券商 API 同步
- 多币种换算、汇率管理
- 多用户家庭共享、多人协作
- 高级报表（按分类占比、资产配置饼图等可作为 v0.2）
- 权限体系复杂化（先默认单用户、基础登录由主项目决定）

## 3. 目标用户与典型场景
### 3.1 目标用户
- 有一定资产分散（存款/基金/股票/贷款/信用卡等），想了解**资产负债表变化**的人。
- 不愿维护复杂账本，但能接受“定期录一次快照”的轻量用户。

### 3.2 核心场景
- 月底/发薪日/大额投资或还款后，快速记录当前资产负债结构。
- 查看净资产是否持续增长、负债是否下降。

---

## 4. 用户故事（User Stories）
1. 作为用户，我希望能快速新增一次“财务快照”，记录当下的资产和负债明细，以便追踪变化。  
2. 作为用户，我希望能给每条明细选择“类型”和“多级分类”，以后能按结构理解自己的资产负债。  
3. 作为用户，我希望能在图表里看到总资产/总负债/净资产随时间的趋势，以便判断财务是否变好。  
4. 作为用户，我希望能查看历史某一次快照的详情，核对当时录入是否准确。  
5. 作为用户，我希望在手机上也能顺畅录入和查看图表。

## 5. 数据模型与字段定义（MVP）
> 建议采用“快照（Snapshot）—明细（Item）—分类（Category）”三层。

### 5.1 财务快照 Snapshot
| 字段 | 类型 | 必填 | 说明/规则 |
|---|---:|---:|---|
| snapshot_id | UUID | 是 | 主键 |
| user_id | UUID | 是 | 所属用户 |
| snapshot_time | DateTime | 是 | 默认当前时间，可手动改为某天某时（精确到日即可，时间可选） |
| total_assets | Decimal(18,2) | 是 | 保存时由明细汇总计算 |
| total_liabilities | Decimal(18,2) | 是 | 保存时由明细汇总计算 |
| net_worth | Decimal(18,2) | 是 | total_assets - total_liabilities |
| currency | String | 是 | MVP 先固定为 CNY（或跟随用户设置） |
| note | String | 否 | 快照备注 |
| created_at / updated_at | DateTime | 是 | 系统字段 |

### 5.2 资产/负债明细 Item
| 字段 | 类型 | 必填 | 说明/规则 |
|---|---:|---:|---|
| item_id | UUID | 是 | 主键 |
| snapshot_id | UUID | 是 | 归属快照 |
| direction | Enum | 是 | `ASSET` / `LIABILITY` |
| item_name | String | 是 | 例如 “招商银行活期”“房贷”“信用卡欠款” |
| amount | Decimal(18,2) | 是 | 非负数；方向由 direction 表示 |
| asset_type / liability_type | Enum | 是 | 见 5.3 |
| category_id | UUID | 否 | 指向叶子分类；允许为空（未分类） |
| category_path | String | 否 | 冗余存储如“资产/金融资产/基金/指数基金”便于展示（保存时生成） |
| note | String | 否 | 明细备注 |
| created_at / updated_at | DateTime | 是 | 系统字段 |

### 5.3 类型枚举（MVP建议）
**资产类型（asset_type）**：
- CASH（现金/零钱）
- DEPOSIT（存款）
- FUND（基金）
- STOCK（股票）
- BOND（债券）
- PENSION（养老金/公积金）
- REAL_ESTATE（房产）
- OTHER_ASSET（其他资产）

**负债类型（liability_type）**：
- CREDIT_CARD（信用卡）
- CONSUMER_LOAN（消费贷/网贷）
- MORTGAGE（房贷）
- CAR_LOAN（车贷）
- PERSONAL_LOAN（亲友借款）
- OTHER_LIABILITY（其他负债）

> 说明：类型用于“快速排序与默认分类提示”；分类用于“用户自定义结构”。

### 5.4 分类 Category（多级）
| 字段 | 类型 | 必填 | 说明/规则 |
|---|---:|---:|---|
| category_id | UUID | 是 | 主键 |
| user_id | UUID | 是 | 所属用户 |
| direction | Enum | 是 | `ASSET` / `LIABILITY`（资产负债分类树分开） |
| name | String | 是 | 分类名（同级不可重名） |
| parent_id | UUID | 否 | 根节点为空 |
| level | Int | 是 | 1~4 |
| sort_order | Int | 否 | 排序 |
| is_leaf | Boolean | 是 | 是否可被明细选择（叶子可选） |
| path_ids | String | 是 | 例如 “root/xx/yy/zz” |
| path_names | String | 是 | 例如 “金融资产/基金/指数基金” |
| created_at / updated_at | DateTime | 是 | 系统字段 |

## 6. 多级分类规则（关键）
### 6.1 层级与选择规则
- 分类深度：**2~4 级**（MVP 默认上限 4，便于未来扩展）。
- 明细只能选择**叶子节点**（`is_leaf=true`）。  
  - 若用户选择了非叶子节点，系统提示“请选择更具体的分类”。
- 资产分类树与负债分类树**独立**（同名允许跨树存在）。

### 6.2 默认分类（建议提供模板，可编辑）
- 资产（示例）：  
  - 金融资产 > 存款 / 基金 / 股票 / 债券  
  - 实物资产 > 房产 / 车辆 / 其他  
- 负债（示例）：  
  - 短期负债 > 信用卡 / 消费贷  
  - 长期负债 > 房贷 / 车贷  

### 6.3 命名与唯一性
- 同一 parent_id 下，name 不可重复（避免 UI 混淆）。
- 允许用户自定义增删改；删除规则见 6.4。

### 6.4 删除/合并规则（MVP最小可行）
- 删除分类时：
  - 若分类下有子分类：不允许删除（提示先删除/移动子分类）。
  - 若分类作为叶子且已被明细引用：不允许直接删除；提示“请先将历史明细改分类或使用‘停用’”（MVP可先做“不允许删除”，后续加停用）。
- 分类调整（移动父节点）后：
  - 新录入使用新路径；历史数据的 `category_path` 可在保存快照时重新生成，或保持冗余可接受不一致（建议：快照保存时重算当前选择路径）。

## 7. 交互流程（MVP）
### 7.1 信息架构（IA）
- 顶部导航（PC）/底部 Tab（移动）：  
  - 首页（快照列表）  
  - 新增快照  
  - 趋势图  
  - 分类管理（可放“设置”里，MVP也可入口放在新增明细页）

### 7.2 新增快照流程
1. 入口：点击“新增快照”
2. 页面结构：
   - 快照时间（默认今天，可改）
   - 两个区块：资产列表、负债列表
   - 每个区块内可“+新增明细”
   - 底部固定：显示实时汇总（总资产/总负债/净资产），按钮“保存快照”
3. 新增明细弹层/页面：
   - 方向（资产/负债）预填
   - 名称、金额、类型（下拉）、分类（分级选择器）、备注
   - 保存后回到快照页（列表新增一条）
4. 保存快照：
   - 校验：至少有 1 条明细（资产或负债均可，建议提示“建议同时记录资产与负债”但不强制）
   - 汇总计算并落库
   - 成功提示，并跳转到“快照详情”或“趋势图”（二选一，建议给 CTA：查看趋势）

### 7.3 快照列表与详情
- 列表项展示：日期、总资产、总负债、净资产、较上次变化（可选，MVP可不做变化）
- 详情页：
  - 顶部：汇总（总资产/总负债/净资产）
  - 明细分组：资产明细、负债明细（可按类型或分类折叠）
  - 操作：MVP建议只做“查看”；编辑/删除快照可放 v0.2（或做删除但需谨慎）

### 7.4 趋势图流程
- 默认进入显示：净资产折线（或三条线同时显示）
- 控件：
  - 时间范围切换：1月/3月/6月/1年/全部
  - 指标切换：总资产/总负债/净资产（或多选显示）
- 图表交互：
  - 点击某个点：展示该快照的汇总值与日期；可跳转快照详情

### 7.5 分类管理（MVP简化）
- 入口：新增明细页的“管理分类”或设置页
- 功能：
  - 查看资产分类树、负债分类树
  - 新增分类：选择父级、输入名称、是否叶子（或默认新增为叶子；有子节点时自动变非叶子）
  - 重命名分类
  - 删除分类（按 6.4 规则）

## 8. 验收标准（Acceptance Criteria）
### 8.1 新增快照
- 能创建快照并持久化：包含 snapshot_time、汇总字段、明细列表。
- 保存时汇总正确：  
  - total_assets = 所有 direction=ASSET 的 amount 之和  
  - total_liabilities = 所有 direction=LIABILITY 的 amount 之和  
  - net_worth = total_assets - total_liabilities
- 明细金额校验：不可为负；为空提示；支持两位小数。
- 资产/负债类型必须选择（若没选，阻止保存并提示）。
- 分类可不选，保存不报错；选择分类必须为叶子节点。

### 8.2 多级分类
- 支持至少 2 级、最多 4 级创建与展示。
- 分类选择器能正确展示层级路径，并回填到明细展示中。
- 同级分类重名被阻止并提示。
- 删除分类遵循规则：有子节点或已被引用时不得删除（MVP实现）。

### 8.3 趋势图
- 当存在 ≥2 次快照时，折线连接顺序正确（按 snapshot_time 升序）。
- 指标切换后，数值与快照汇总一致。
- 时间范围筛选正确（边界日期包含规则明确：包含起止日）。
- 无快照时展示空状态：提示去新增快照。

### 8.4 兼容性与体验
- 移动端可滚动、底部汇总条不遮挡输入；PC端布局不变形。
- 关键操作（保存快照）防重复提交（loading + 幂等）。

## 9. 埋点建议（Analytics）
> 目标：定位“录入是否顺畅”“趋势图是否被使用”“分类是否成为阻碍”。

### 9.1 核心漏斗
- `snapshot_create_enter`（进入新增快照页）
- `snapshot_item_add_click`（点击新增明细）
- `snapshot_item_save_success` / `snapshot_item_save_fail`（明细保存成功/失败，带错误码）
- `snapshot_save_click`
- `snapshot_save_success` / `snapshot_save_fail`（带失败原因：校验失败/网络/服务端）
- `trend_view_enter`
- `trend_metric_switch`（资产/负债/净资产）
- `trend_range_switch`（1m/3m/6m/1y/all）
- `trend_point_click`（点击图表点，携带 snapshot_id）

### 9.2 分类相关
- `category_manage_enter`
- `category_create_success/fail`（fail带原因：重名/超层级/非法字符）
- `category_delete_attempt`（携带是否被阻止、阻止原因：has_children/has_reference）
- `category_select_open`、`category_select_success`（携带层级深度 level）

### 9.3 关键属性（Properties）
- 用户端：device_type（mobile/desktop）、viewport、language
- 快照：items_count_asset、items_count_liability、total_assets、total_liabilities（可做分桶以保护隐私）
- 分类：selected_level、direction

## 10. 非功能需求（NFR）
### 10.1 性能
- 首屏加载（有缓存情况下）< 2s（移动网络 4G 下可接受 < 4s）。
- 快照列表 1,000 条以内滚动不卡顿（分页/虚拟列表可选）。
- 图表渲染：点数 ≤ 500 时交互流畅；超过则抽样或按月聚合（未来优化点，MVP可先限制）。

### 10.2 安全与隐私
- 全站 HTTPS。
- 敏感数据（金额等）在日志中脱敏；埋点金额建议做区间化或不采集原值。
- 基础防护：CSRF/XSS；服务端校验不可依赖前端。

### 10.3 可靠性
- 保存快照接口幂等：同一 request_id 防止重复插入。
- 客户端断网提示与重试（至少提示“网络异常，请重试”）。

### 10.4 可用性
- 表单错误就地提示（字段下方），保存按钮在移动端可触达。
- 空状态明确引导下一步操作。

## 11. 风险与取舍（MVP决策点）
- **编辑/删除快照**：MVP建议先不做，降低数据一致性复杂度；必要时仅支持删除最近一次快照并二次确认。
- **分类删除/停用**：MVP用“禁止删除已引用分类”兜底；v0.2 增加“停用”更符合用户预期。
- **多币种**：MVP单币种，避免汇率与展示混乱。
