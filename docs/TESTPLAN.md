# 个人财务记录 Web（手机/电脑）MVP 测试方案

> 覆盖 2 个核心功能：
> 1) 资产/负债记录 + 多级分类管理
> 2) 趋势图（资产/负债/净资产趋势，按时间与分类筛选/对比）

## 1. 背景与范围
**MVP 功能（2 个）：**
1) 资产/负债记录 + 多级分类管理（新增/编辑/删除/列表/筛选/汇总）  
2) 趋势图（按时间维度展示资产、负债、净资产趋势；按分类筛选/对比）

**不在本轮范围：** 登录注册、预算、账单导入、多人协作、复杂报表导出等。

---

## 2. 测试目标
- 核心业务正确：资产/负债金额与分类结构正确、增删改查可靠
- 趋势图口径一致：同一数据在列表汇总与图表一致，时间区间/分类筛选一致
- 移动端可用：小屏交互不崩、关键路径顺畅
- 数据一致性：并发、刷新、跨端一致、删除引用规则明确
- 基本性能与安全：常用操作响应可接受、无明显安全缺陷

---

## 3. 测试环境与数据准备（建议）
- 环境：测试/预发各一套（含独立数据库）
- 浏览器/系统：
  - iOS：Safari（近 2 个大版本）
  - Android：Chrome（近 3 个大版本）
  - PC：Chrome/Edge（最新版）、Safari(macOS)（可选：Firefox）
- 基础数据：
  - 多级分类树（≥3 层，含同名不同层、空子类、深层叶子）
  - 资产/负债记录（跨月、跨年、同日多笔、大数、长备注）
  - 边界数据（0、极大值、长文本、特殊字符、emoji）

## 5. 边界与异常测试（重点清单）
### 5.1 数值与字段边界
- 金额：0、0.01、小数位上限、极大值（如 10^12）、前后空格、千分位
- 负数：通常不允许（负债靠类型/方向表达）
- 日期：跨年、未来日期是否允许、闰日 2/29
- 文本：超长、特殊字符（< > ' " &）、emoji
- 分类层级：空分类、叶子/非叶子可选规则、最大层级、同级重名

### 5.2 异常/失败
- 网络断开/超时：保存中断、重复点击保存、重试（避免重复插入）
- 4xx/5xx：提示友好、表单数据不丢
- 并发编辑：两端同时改同一条记录（按策略验证：覆盖/冲突提示）
- 删除引用：删除被记录引用的分类（阻止/迁移策略一致且可追溯）

## 6. 兼容性测试（移动端重点）
- iOS Safari：小屏 + 大屏
- Android Chrome：低中端 + 高端
- PC：Windows Chrome/Edge；Mac Safari/Chrome

重点检查：响应式布局、触控命中面积、输入法/数字键盘、滚动与图表交互冲突、刘海屏安全区。

---

## 7. 数据一致性与口径校验
必须固化为断言：
- 净资产 = 资产合计 - 负债合计（同一筛选条件/时间范围）
- 父类分类筛选 = 父类 + 所有子类
- 趋势图聚合与列表聚合一致（粒度边界：日/周/月定义一致）

---

## 8. 简单性能测试（轻量）
- 前端：首屏可交互 4G < 3s；趋势图 1k 点以内 < 1s（基线）
- 后端：新增记录 P95 < 300ms；列表/趋势聚合 P95 < 500ms（视环境调）

方法：DevTools 取样；构造历史记录 1w 条，查询近 30/90/365 天。

---

## 9. 基本安全检查（MVP 必做）
- 鉴权：未登录访问 record/category/trend API 应 401/403
- 越权/IDOR：用户 A 不能访问/修改/删除用户 B 的数据
- XSS：备注/分类名含脚本不执行（tooltip/详情都要转义）
- CSRF：若 Cookie Session 则必须有 CSRF；Token 模式检查存储与同源
- 输入校验：金额/日期/分类 ID 后端强校验；参数化防注入
- 敏感信息：响应不泄露调试栈/SQL 等

P0：创建/更新/删除记录；分类 CRUD + “引用删除策略”；趋势接口粒度/边界日断言；越权用例。
